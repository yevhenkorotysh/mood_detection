Mood Detection Service from Audio Files

This application is aimed at building a Mood Detection Service from audio files, offering two primary services:

  1. Service to Store an Audio File: This service enables users to upload an audio file, which is then securely stored in a cloud storage system, like AWS S3.
  2. Service to Retrieve Mood: The second service analyzes the uploaded audio file to determine the mood and returns one of three different moods: Happy, Sad, or Calm. The actual mood determination is not implemented in this project; instead, a random mood generator is used.

Architecture

Language Choice

The service code is implemented with Python/Django due to its extensive libraries and ease of use for audio processing.

Components

The architecture of the project involves several key components:

  1. Audio File Storage: Cloud storage, such as AWS S3, is used to store uploaded audio files. These files should be limited to a maximum duration of 30 seconds and can be in WAV or MP3 format only.
  2. Audio Processing: To analyze the mood, the uploaded audio file is processed. In this example, a random mood is generated. However, a real implementation would require audio analysis using libraries like Librosa for feature extraction and AI model training.
